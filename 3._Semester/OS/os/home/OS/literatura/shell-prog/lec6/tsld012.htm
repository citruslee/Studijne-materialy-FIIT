Some Appropriateness Issues

  • In Unix it more than one way and one tool can usually be found to do what one wants to do

  • We saw few simple cases where sed used like head, grep, and tr

  • Generally the simpler the program the faster and more efficiently it will run

    • for example tr just translates characters so it will do this faster than other utilities such as sed

  • When a program is written to do something specific it will generally do that more efficiently than a general purpose program

    • grep function is to look for things within files
    • sed used in this way has more overhead therefore less efficient

  • Should always choose the most appropriate tool for the total job
