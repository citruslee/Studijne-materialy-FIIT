sed general syntax

  • syntax: sed command file

    • command is the editor command to be applied to each line (generally enclosed in single quotes to hide from shell
    • file is the input file
    • failure to include file means that sed will work on the standard input

  • For example: sed ‘s/UNIX/UNIX(TM)/g’ * > zzz

    • substitues UNIX(TM) for UNIX
    • substitution is global not just first encountered occurrence on line that is the normal behavior of command
    • shell expands * to the actual name of all files
    • substitution doesn’t happen in input file but just on output
