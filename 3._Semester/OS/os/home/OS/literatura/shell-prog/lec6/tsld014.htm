sed Summary

  • sed can do much more than just the type of things given in the examples

    • branching
    • testing conditions
    • looping
    • remembering previous lines

  • Most frequent uses, however, are like the examples that we provided

  • sed is convenient

    • because it will handle arbitrarily long inputs
    • is fast
    • syntax similar to ed with regular expressions and line at a time processing

  • limitations

    • provides limited form of memory (hard to remember from line to line)
    • makes only one pass over the data
    • not possible to go backwards
    • not possible to make forward references
    • no facilities for manipulating numbers
