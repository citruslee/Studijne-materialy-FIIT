Files and their Inodes

  • A file has several components: name, contents, and administrative info (e.g. permissions and modification times)

  • Administrative information is stored in the inodes

    • inodes also contain contain essential system data: how long, where on disk stored, etc.
    • three times in inode: time contents last modified, time file last used, time that inode itself was last changed (e.g. set permissions)

  • Directory actually only stores the starting inode number of files within it

  • Unix tracks files only by their inode numbers

    • Unix knows nothing at the file system level about names
    • user assigned names are in the directory associated with the inode that is that file as far as Unix is concerned.
