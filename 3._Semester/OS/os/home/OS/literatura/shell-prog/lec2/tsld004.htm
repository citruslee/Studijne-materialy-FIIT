File System - Permissions

  • Every file has a set of permissions associated with it which determine what specific groups can do with file

    • directory permissions affect what can be done with files. If make a directory non readable then all files in it are non-readable
    • superuser (system administrator) authority overrides ALL individual permissions
    • default set of permissions which are applied when file created; can change via environment variable

  • Permission categories are : read, write, execute

  • Permission groups are

    • owner (u-user)
    • group (g)
    • others (o)

  • Permissions are set at the file level

  • Permissions permit only a modest security mechanism and Unix enforcement of these is not real strict
