Inodes (contd)


  • Names are a feature of directory and not part of file.

  • System knows files only by inodes

  • links are handled by using the same inode number rather than actually copying file

    • in directory there is the original name with its inode
    • in directory there is the link name with same inode
    • lookup of name in directory gets the inode

  • only when the last link to a file disappears does the system remove the inode and therefore the file itself

  • inode gives only starting location (1st cluster of file) on device;

    • pieces of the file do not have to be contiguous
    • each piece is linked to the next through a pointer to the location of the next cluster for the file

  • fsck utility run at boot up checks the file system and one of the functions it does is ensure that the inodes are valid
