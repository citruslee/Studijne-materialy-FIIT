File or File Related Metacharacters

  • shell recognizes a number of characters as special known as metacharacters- as it processes a command line

  • metacharacters require special action by shell before passing values to the command

  • * is the most commonly used metacharacter

    • tells shell to search directory for filenames in which any string of characters occurs in the position of asterisk (zero or more characters)
    • shell must expand before passing to the command (like echo) or displaying on output so * gets replaced by all the filenames

  • ? similar to * but looks only for a single character at the position

  • & requires shell to run a command in background rather than foreground

  • | sends output of one command to input of next

  • > < >> << are redirection symbols (change location of stdin, stdout, stderr files)

  • ; separates commands so can put more than one command on line before <>

  • \ continues a command line

  • ë quote with no metacharacter replacement ì quote with metachar replacement
