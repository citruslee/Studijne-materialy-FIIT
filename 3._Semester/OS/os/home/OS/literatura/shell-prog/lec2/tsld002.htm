Overview

  • Unix does not track file types like many operating systems (e.g., VMS and RMS files)

  • All files, even directories, are just a stream of bytes

  • This approach facilitates flexibility in the commands (allows commands to be used on files not file types)

    • allows some non meaningful commands (e.g. sort /bin/ed)
      • this seems to be asking to sort the line editor ed
      • when sort does this the output is not meaningful but sort and most Unix commands donít preclude doing this
      • user makes determination of what is meaningful and what isnít

  • Many commands can operate on any file and there is no good reason for Unix to restrict their capabilities

  • Leaves specific reactions to specific commands to be based on the purpose determined by the creator or the command rather than the operating system

  • Instead of creating distinctions Unix tries to efface them whether in files or commands
