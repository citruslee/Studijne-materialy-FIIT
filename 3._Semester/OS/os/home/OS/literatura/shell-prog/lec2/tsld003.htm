Overview (contd)

  • Such uniformity in file systems is unusual in operating systems

  • most systems require negotiation by program or user to create a particular file type

    • this approach makes files operating system dependent
    • reduces flexibility and portability

  • However implementation limits often exist in programs dealing with text which often have limits on the number of bytes they can handle before encountering the newline

  • Everything in storage is a file

  • Even a directory is just a file

    • directory is a file that the shell understands and uses
    • in Unix possible for program (mostly in C) to also read and use the directory file
    • Unix also provides some commands that provided directory file information
